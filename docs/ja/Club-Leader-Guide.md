# クラブリーダー向けガイド

このガイドは、英語版を正本として同じ構成で作成しています。

## アクセスと役割

クラブリーダーは、ホームと参加可否を利用できます。さらに、当日の管理ダッシュボードを確認し、訪問予定を管理できます。保護された各処理では、サーバー側でも権限が再確認されます。

クラブリーダーは、活動・イベント、メンバー、ロール、設定を管理できません。

## 生徒向け機能

ホームと参加可否は、生徒向けガイドと同じ方法で利用します。ナビゲーションでは、生徒専用の自分の訪問スケジュールではなく、訪問予定管理が開きます。クラブリーダーも訪問担当に割り当てることができ、自分の担当も管理一覧に表示されます。

## ダッシュボードの利用

1. **ダッシュボード**を開きます。
2. 本日実施される有効な活動が複数ある場合は、対象の活動を選びます。
3. 参加可否と訪問担当の集計およびメンバー区分を確認します。
4. `Unavailable`と回答した訪問担当者の警告一覧を確認します。
5. 読み取り専用の出席集計を確認します。

ダッシュボードには、クラブ全体の参加可否、参加可否の個人メモ、訪問担当情報、出席情報が含まれます。これらは機密情報として扱い、閲覧権限のない生徒と共有しないでください。

ダッシュボードが読み込むのは、本日実施される有効かつ中止されていない活動だけです。

## 訪問予定の管理

「訪問予定」は、Volunteer Assignmentsを画面上で表す名称です。

1. **訪問予定管理**を開きます。
2. 有効な今後の活動を選びます。
3. 現在の担当と、対象メンバーごとの現在の参加可否を確認します。
4. 在籍中の生徒またはクラブリーダーを1名以上追加します。
5. 必要に応じて、活動内容、場所、出発時刻、担当状態、メモを入力します。
6. 担当を保存します。

同じSession IDとMember IDの組み合わせによる重複担当は防止されます。`Unavailable`と回答した人を担当にする場合は、明示的な確認が必要です。`Unsure`または未回答の場合は通知が表示されますが、保存は拒否されません。

出発時刻は有効な時刻で、活動の終了時刻より後には設定できません。移動時間を考慮し、活動開始時刻より前に設定することはできます。

## 担当の編集・取消・削除

- 今後の担当は、管理対象の項目を編集できます。
- 履歴を残す必要がある担当には`Cancelled`を使用します。
- 完全削除は、安全に削除できる今後の担当だけに制限されます。
- 本日の担当、過去の担当、出席履歴がある担当、または対応する活動が見つからない担当は完全削除できません。
- 訪問予定の操作によって、関連する出席履歴が削除されることはありません。

## 言語の変更

ページ上部の言語セレクターを使用します。画面ラベル、日付、警告、状態は日本語または英語で再描画されます。氏名、活動名、活動内容、場所、メモは翻訳されません。

## トラブルシューティング

- ダッシュボードに活動が表示されない場合は、本日の日付で、有効かつ中止されていない活動があるか確認してください。
- メンバーを選択できない場合は、そのメンバーが在籍中で、ロールが`Student`または`Club Leader`であるか教員に確認してください。
- 削除が拒否された場合は、履歴を保持するため担当を取り消してください。
- クラブリーダーが管理できない活動情報やメンバー情報の修正は、教員に依頼してください。
